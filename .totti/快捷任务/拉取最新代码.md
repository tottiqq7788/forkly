# 任务：拉取 Forkly 最新代码

## 目标

从 GitHub 拉取 Forkly 项目的最新代码，并保护当前本地改动不被覆盖。

## 项目信息

- 项目名称：forkly
- 项目路径：`/Users/totti/个人/project/mod/forkly`
- 默认分支：`main`
- 远程名称：`origin`
- 远程地址：`https://github.com/tottiqq7788/forkly.git`
- 平台：GitHub

## 执行流程

1. 进入项目根目录并检查状态：

   ```bash
   pwd
   git status --short --branch
   git remote -v
   ```

2. 如果远程仓库未配置，使用：

   ```bash
   git remote add origin https://github.com/tottiqq7788/forkly.git
   git branch -M main
   ```

   如果远程地址不是 `https://github.com/tottiqq7788/forkly.git`，先说明差异，不要擅自覆盖。

3. 如果存在本地未提交变更：

   - 先展示变更列表和统计：

     ```bash
     git status --short
     git diff --stat
     ```

   - 不要丢弃本地变更。
   - 需要拉取时，先暂存现场：

     ```bash
     git stash push -u -m "拉取前自动暂存：$(date '+%Y-%m-%d %H:%M:%S')"
     ```

4. 拉取远程更新：

   ```bash
   git fetch origin
   git log --oneline --decorate --graph --left-right main...origin/main
   git pull --no-rebase origin main
   ```

5. 拉取完成后检查依赖：

   ```bash
   go mod tidy
   cd web && npm install && cd ..
   ```

6. 运行验证：

   ```bash
   go test ./...
   cd web && npm run build && cd ..
   ```

7. 展示最终状态：

   ```bash
   git status --short --branch
   git log --oneline --decorate -5
   ```

## 输出要求

- 全程使用中文。
- 如果拉取时发生冲突，停止并列出冲突文件，不要执行 reset、checkout 或丢弃操作。
- 如果本地变更被 stash，结束时说明可用 `git stash list` 和 `git stash pop` 恢复。

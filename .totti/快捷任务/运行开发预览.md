# 任务：运行 Forkly 开发预览

## 目标

启动 Forkly 的开发预览环境（Vite + 本地 API），不依赖菜单栏图标，也不需要先打包 DMG。

## 项目信息

- 项目名称：forkly
- 项目路径：`/Users/totti/个人/project/mod/forkly`
- Web 目录：`web`
- 本地预览地址：`http://127.0.0.1:5173/`
- 本地 API：`http://127.0.0.1:8787/`（Vite 会把 `/local-api` 代理到这里）
- 远程地址：`https://github.com/tottiqq7788/forkly.git`

## 启动方式（推荐：双终端）

### 终端 1：本地 API（开发模式）

```bash
cd /Users/totti/个人/project/mod/forkly
FORKLY_DEV=1 go run ./cmd/forkly
```

或：

```bash
make preview-api
```

`FORKLY_DEV=1` 会：

- 不启动菜单栏
- 固定监听 `127.0.0.1:8787`
- 开放 `POST /local-api/v1/session/dev-login`，供 Vite 页面自动建会话

### 终端 2：Vite

```bash
cd /Users/totti/个人/project/mod/forkly
make preview-web
```

或：

```bash
cd /Users/totti/个人/project/mod/forkly/web
npm install
npm run dev -- --host 127.0.0.1
```

### 浏览器

打开：

```text
http://127.0.0.1:5173/
```

开发环境下前端会在无会话时自动调用 `dev-login`，无需菜单栏「打开控制台」。

## 仅看静态界面

若暂时不需要真实 API，只跑 Vite 也可以；页面会提示「本地 API 未启动」。

## 注意事项

- 正式 App / 生产构建仍须从菜单栏 claim 会话；`dev-login` 仅在 `FORKLY_DEV=1` 时可用。
- 不要用开发预览替代 macOS 打包验证。

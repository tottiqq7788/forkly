## 审查重点

1. 功能正确性：

   - 本地 API 会话、CSRF、Origin、Host 校验是否可靠。
   - Git 状态、差异、提交、历史解析是否处理特殊路径、中文、空格和空仓库。
   - 保存版本是否只提交用户选中的文件。
   - 异常状态是否会阻断高风险操作，而不是自动破坏性修复。

2. 安全与数据保护：

   - 不接受任意绝对路径读取。
   - 符号链接不能逃逸项目根目录。
   - 日志不记录密钥、Token、文件正文或敏感路径凭据。
   - 不跳过 Git hooks，不修改用户全局 Git 配置。

3. 工程质量：

   - Go 包职责是否清晰。
   - 前端组件是否过大、状态是否混乱。
   - 是否存在死代码、重复逻辑、过期脚本或错误的远程地址。
   - 测试是否覆盖关键路径。

4. 发布风险：

   - macOS 打包脚本是否正确处理内置 Git、双架构、签名和公证。
   - `.gitignore` 是否排除了 `dist/`、`third_party/git/`、缓存和依赖目录。
